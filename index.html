🐙 Deploy to GitHub Pages — Share with your team

Host this file on GitHub Pages for free. Anyone on your team opens the URL, enters the sheet URL once, and sees the live map — no install, no backend.

1
Go to github.com/new → create a new public repository named e.g. release-dashboard
2
Click Add file → Upload files → drag this release-intelligence-live.html file → commit
3
Go to Settings → Pages → Source: Deploy from a branch → Branch: main → folder: / (root) → Save
4
Rename the file to index.html before uploading (or add a second upload). GitHub Pages serves index.html at the root URL automatically.
5
Your team URL will be: https://YOUR-USERNAME.github.io/release-dashboard/ — share this link. Each person enters the sheet URL once; it's saved in their browser.
6
To update the tool in future: upload the new HTML file to the same repo and commit. GitHub Pages auto-deploys in ~30 seconds.
✅  Zero cost — GitHub Pages is free for public repos
✅  No server — runs entirely in the browser
✅  Auto EOD sync — each viewer's browser refreshes at 18:00 their local time
✅  Sheet URL is saved in localStorage per browser — set once, always remembered

📜 Apps Script — Real-time Push (Optional)

The EOD sync is automatic. If you want instant updates the moment someone marks a release as "Released" in the sheet, paste this into Google Apps Script and deploy as a web app.

// ReleaseIntel — Google Apps Script // Extensions → Apps Script → paste → Save → Deploy → Web App function doGet(e) { const ss = SpreadsheetApp.getActiveSpreadsheet(); const sheet = ss.getActiveSheet(); const data = sheet.getDataRange().getValues(); const headers = data[0]; const rows = data.slice(1).map(row => { const obj = {}; headers.forEach((h, i) => { obj[h] = row[i]; }); return obj; }); const payload = JSON.stringify({ source: 'google-apps-script', updated: new Date().toISOString(), rows }); return ContentService .createTextOutput(payload) .setMimeType(ContentService.MimeType.JSON); } function onEdit(e) { Logger.log('Row edited: ' + JSON.stringify(e.range.getValues())); }
After deploying, paste the Web App URL (not the sheet URL) into the Setup tab — the tool will fetch from your script endpoint instead of the published CSV, giving you real-time data on every Sync Now click.
